# 🎰 体彩推荐分析系统

> 基于近 2 年历史数据，为 **超级大乐透、排列3、排列5** 提供科学综合打分推荐的本地 Web 应用。

---

## ⚠️ 重要免责声明（请务必阅读）

**彩票每次开奖均为独立随机事件，任何算法都无法真正预测中奖号码。**

本系统仅基于历史统计（频率、遗漏、趋势、组合分布）做"最符合历史规律"的参考推荐，
**仅供娱乐与研究学习，不构成任何投注建议，也不保证任何收益。**
官方（中国体育彩票）已多次公开辟谣"AI 预测彩票"。请理性购彩，量力而行，切勿沉迷。

## ✨ 功能特性

- **三种玩法**：超级大乐透（前区5/35 + 后区2/12）、排列3（3位）、排列5（5位）。
- **每日刷新**：在 Web 前端一键"刷新数据并计算推荐"，自动抓取最新开奖并增量入库。
- **科学综合打分**：综合 出现频率 / 遗漏比 / 近期趋势 三维度加权评分（权重可在 `config.py` 调）。
- **每种玩法 5 组单选 + 小/中复式**：
  - 单选 5 组各有明确选号论点：均衡热号、冷号回补、趋势加速、组合合规、幸运组合。
  - 大乐透：小复式（前6+后3，12注）、中复式（前8+后4，112注）。
  - 排列3/5：每位 2 个或 3 个候选号。
- **可读推荐理由**：每组推荐附"为什么推荐"，引用具体统计数据（频率、遗漏、和值区间等）。
- **可视化**：号码频率柱状图、遗漏值条形图、冷热标签（红=热/蓝=冷/灰=平）。
- **历史数据**：按玩法/期号/日期搜索的分页表格。
- **命中回测**：将最新推荐与实际开奖对比，诚实展示命中率。
- **轻量鉴权**：登录密码保护，默认 `admin123`（可在 `config.py` 改）。

## 🧮 推荐算法说明（v2 增强版）

### 评分公式

对每个候选号码计算综合分（v2 增强版）：
```
综合分 = 频率权重 × (0.6×EWMA频率分 + 0.4×贝叶斯平滑频率分)
       + 遗漏权重 × 遗漏分位分
       + 趋势权重 × 0.6 × 趋势分
```

v2 改进点（基于回测反馈）：
- **EWMA 频率**（指数加权）：近期权重更高但平滑衰减，比"近30期硬切"更稳健
- **贝叶斯平滑频率**：`freq = (出现次数+C) / (总期数+C×号池大小)`，抗小样本噪声
- **遗漏分位**：当前遗漏在历史遗漏段分布中的百分位，比"遗漏比"更抗极值
- **趋势降权**：趋势信号噪声大，从 0.25 降至 0.15（0.25×0.6）

### 5 组单选策略

| 组 | 策略 | 含义 |
|---|---|---|
| 1 均衡热号 | 综合分最高 | 最贴合近期主流规律 |
| 2 冷号回补 | 偏重高遗漏比 | 追"该出"的冷号 |
| 3 趋势加速 | 偏重 momentum | 追升温号 |
| 4 组合合规 | 和值/奇偶/大小落历史高频区间 | 符合组合规律 |
| 5 幸运组合 | 高分池随机 | 带随机性的组合 |

> 排列3/5 为数字型，按每个位置独立打分（每位 0-9 分布）。组间有**多样性惩罚**（15%降权已选号），避免5组推荐高度雷同。

## 🔬 模型有效性验证（重要！）

本系统内置了**逐期回放回测引擎**，在"命中回测"页面点击"运行验证"即可查看。

### 回测方法
对每个历史测试点 t：
1. 只用 t 之前的数据计算统计特征（**避免未来泄漏**）
2. 用当前模型在 t 处生成 top-K 候选号
3. 与 t 期实际开奖对比，记录命中数
4. 与"纯随机基线"（从号池随机选同数量号的期望命中率）对比

### 回测结果（119个测试点/玩法）

| 玩法 | 模型命中率 | 随机基线 | 提升 | 结论 |
|---|---|---|---|---|
| 超级大乐透 | 14.05% | 14.29% | **-1.7%** | ≈ 随机 |
| 排列3 | 8.96% | 10.00% | **-10.4%** | ≈ 随机 |
| 排列5 | 10.92% | 10.00% | **+9.2%** | ≈ 随机 |

**诚实结论：模型命中率与纯随机基线无显著差别（±10% 以内）。这完全符合彩票独立随机的数学本质——任何模型都无法可靠预测中奖号码。**

v1 模型（纯频率/遗漏）在大乐透上曾达到 **-15%**（比随机还差，原因是过度追逐热号）。v2 通过 EWMA+贝叶斯平滑+多样性约束修复了这个问题，使模型至少不再"反向"，但仍无法战胜随机。

## ❓ 常见问题

**Q：能保证中奖吗？**
A：**绝对不能。** 彩票是纯随机事件，本系统只提供统计参考。回测已证明模型命中率≈随机基线（详见"命中回测"页的"模型有效性验证"）。

**Q：数据多久更新一次？**
A：你点一次"刷新"就更新一次。排列3/5 每日开奖，大乐透周一/三/六开奖。

**Q：抓取失败怎么办？**
A：500.com 偶尔会限流。稍等几分钟后重试即可；失败信息会显示在首页"最近刷新记录"。

**Q：如何修改密码或调权重？**
A：编辑 `config.py` 后重启 `python app.py`。
